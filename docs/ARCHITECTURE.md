# Архитектура DreamCalc

## Общая структура

```
DreamCalc/
├── src/
│   ├── index.html          # Точка входа
│   ├── styles/
│   │   └── main.css       # Стили поверх Bootstrap
│   └── scripts/
│       ├── app.js         # Главный модуль (контейнер)
│       ├── calculator.js  # Бизнес-логика
│       ├── uiComponents.js # UI-компоненты
│       ├── dreamData.js   # Данные целей
│       └── utils.js       # Утилиты
```

## Принципы архитектуры

### 1. Модульность
- Каждый модуль имеет чёткую ответственность
- Минимальная связность между модулями
- ES6 модули для импорта/экспорта

### 2. Состояние приложения
- Централизованное состояние в `app.js`
- Односторонний поток данных
- UI реагирует на изменения состояния

### 3. Разделение логики и представления
- `calculator.js` — чистая бизнес-логика (без DOM)
- `uiComponents.js` — чистое представление (без состояния)
- `app.js` — связующий слой

### 4. Масштабируемость
- Легко добавлять новые типы целей
- Можно расширять расчёты (инфляция, инвестиции)
- Подготовлено к добавлению роутинга

## Поток данных

1. Пользователь выбирает цель → `app.selectDream()`
2. Форма автозаполняется → `app.autofillForm()`
3. Пользователь вводит данные → состояние обновляется
4. Нажатие "Рассчитать" → `app.handleCalculate()`
5. Вызов `calculator.calculatePlan()` → возврат результатов
6. Рендеринг результатов → `app.renderResults()`

## Управление зависимостями
- Bootstrap через CDN (сетка, базовые компоненты)
- Кастомные стили поверх Bootstrap
- Vanilla JavaScript ES6+ (без фреймворков)
- Все зависимости явно импортируются
